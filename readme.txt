2、Android中主要资源文件及文件夹介绍：
（1）Android的主要开发文件夹是app文件夹，其余的文件夹大部分都是自动生成的文件夹以及文件。
（2）build文件夹主要是用来存放编译项目时生成的文件以及apk，libs文件夹主要使用来存放引用的第三方的库，src文件是主要用来编写代码，存放图片以及其他的资源文件。

3、mipmap和drawable文件夹的关系与区别。
    在更新了Android-studio以后发现创建项目的时候，原来的drawable文件夹都被替换为mipmap文件夹了，就查询了网上的文档。
    （1）mipmap文件夹：
        mipmap一般只存放Launcher时的图片，应用程序会选择最佳的分辨率图标来显示到主屏幕中
        如果需要变大变小变大变小的，有动画的，放在mipmap中能有更高的质量。
        引用方式：XML中：android:icon="@mipmap/ic_launcher" 代码中：R.mipmap.ic_launcher
    （2）drawable文件夹
        drawable文件夹一般存放尺寸固定的，不需要更改的图片。支持的图片类型：PNG、JPEG、GIF、9-Patch以及自定义的按压、正常、获取到焦点的xml
        引用方式：XML中：android:icon="@drawable/ic_launcher" 代码中：R.drawable.ic_launcher
     总结：
        mipmap 只是放 ic_launcher ，其他的图片放在 drawable中，如果非要放到mipmap中也不会有什么其他的影响。
4、Android资源文件命名规范
    1、如何查看Android中包含哪些资源文件夹呢？
    右键res文件夹，选择Android Resource director，点击resource type就可以查看到Android包含的所有的资源文件夹了。
    2、为什么要说Android资源文件夹的命名规范呢？
    因为Android中可以根据国家、系统版本、屏幕分辨率、屏幕尺寸、屏幕像素密度之类的来创建资源文件夹，方便在不能场景中使用不同的资源文件。
    4、创建Android资源文件夹
    咱们可以自己直接创建一个普通的文件夹，不通过选择Androud Resource director的方式
    创建方式：<resources_name>-<config_qualifier>
    （1）<resources_name>：资源文件夹名字
    （2）-：连接符，连接多个配置限定词
    （3）<config_qualifier>：配置限定词。国家简称，系统版本，屏幕尺寸等。
    （4）限定条件必须按照排列顺序排列，如果排列顺序不对则编译不通过。
